# Test Categories

## Schema tests
Validate that required AWC declaration sections remain present and protocol-owned structure is enforced.

## Behavior tests
Exercise blocked, approval-gated, escalated, and budget-exhausted behavior through the reference runtime flow.

## Audit tests
Confirm that decisions produce audit artifacts with required fields, event types, and outcomes.

## Reporting tests
Confirm that runtime-generated daily reports remain protocol-shaped and that compliance reports contain the fields required by this suite.

## Self-hosted onboarding tests
Confirm that a self-hosted worker declaration exposes minimal `metadata.platform_hints` information and that mock invocation/healthcheck artifacts match the declared config shape.
