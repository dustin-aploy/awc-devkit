# Certification Logic

The AWC test suite treats certification as a repeatable evidence process rather than a self-assertion.

A certified result requires:

- use of protocol-owned schemas and versions;
- execution of the full reference check catalog;
- zero failed checks; and
- a generated compliance report that captures the run outcome.

This repository provides the foundation for that process. A future registry can consume the resulting reports, but the registry does not replace the suite itself.
